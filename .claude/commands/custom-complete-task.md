現在進行中のタスクを完了します。

以下の手順で作業してください：

## 1. 進行中タスクの確認

docs/PLAN.md を開き、ステータスが `[→]` のタスクを見つけてください。
タスク番号と内容をメモしてください（例: `1.1.1 Dockerfile 作成`）。

## 2. 完了確認

タスクの成果物が以下を満たしているか確認してください：

- [ ] 要件定義に沿った実装になっている
- [ ] テストが必要なタスクの場合、テストを作成・実行した
- [ ] エラーなく動作する

## 3. ステータス更新

タスクのステータスを `[→]` から `[x]` に更新してください。

## 4. 変更をコミット

以下のコマンドでコミットしてください：

```bash
git add .
git commit -m "feat: タスク {番号} {タスク名} 完了"
```

**コミットメッセージの例：**
- `feat: タスク 1.1.1 Dockerfile作成 完了`
- `feat: タスク 1.2.3 execute_code ツール実装 完了`
- `docs: タスク 2.1.10 サンプルカタログ作成 完了`

**type の使い分け：**
- `feat` - 機能実装タスク
- `docs` - ドキュメントのみのタスク
- `test` - テスト作成タスク
- `fix` - バグ修正タスク

## 5. 次のタスクの提案

同じフェーズ内で、次に着手すべきタスク（ステータスが `[ ]` のもの）を提案してください。
依存関係がある場合は考慮してください。

## 6. 完了報告

以下の形式で報告してください：

```
## タスク完了

- タスク: {番号} {タスク名}
- コミット: {コミットメッセージ}
- 次のタスク候補: {番号} {タスク名}
```
