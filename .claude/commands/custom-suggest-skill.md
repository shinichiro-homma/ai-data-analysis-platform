現在の開発状況を分析し、Skill化すべき候補を提案します。

以下の手順で作業してください：

## 1. 分析対象の確認

直近の作業で以下のパターンを探してください：

### 外部ドキュメントへの繰り返しアクセス

- 公式ドキュメントを何度も参照している技術
- 毎回同じ検索をしているトピック
- エラー解決のために繰り返し調べている内容

### 繰り返し実装しているパターン

- 複数のファイルで同じ構造を持つコード
- 新規ファイル作成時に毎回同じボイラープレート
- コピー&ペーストしているコードパターン

### 暗黙知になっている手順

- 特定の操作を行う際の定型手順
- 環境構築やデバッグの手順
- トラブルシューティングの知見

## 2. Skill化の評価基準

各候補について、以下の基準で評価してください：

| 基準 | 説明 | 重み |
|------|------|------|
| **繰り返し頻度** | どのくらいの頻度で参照/実装するか | 高 |
| **複雑さ** | 毎回思い出すのが難しいほど複雑か | 高 |
| **安定性** | 内容が頻繁に変わらないか | 中 |
| **汎用性** | このプロジェクト以外でも使えるか | 低 |

**Skill化すべき条件:**
- 繰り返し頻度が高い AND 複雑さが高い
- または、毎回外部ドキュメントを参照している

## 3. 既存 Skill の確認

`.claude/skills/` ディレクトリを確認し、既存の Skill と重複しないか確認してください。

```bash
ls -la .claude/skills/
```

## 4. 提案の作成

以下の形式で Skill 化の候補を提案してください：

```
## Skill化の提案

### 候補1: {Skill名}

**対象:** {技術/パターンの名前}

**Skill化の理由:**
- {理由1}
- {理由2}

**含めるべき内容:**
- {内容1}
- {内容2}

**評価:**
- 繰り返し頻度: 高/中/低
- 複雑さ: 高/中/低
- 安定性: 高/中/低

**推奨度:** ⭐⭐⭐⭐⭐ / ⭐⭐⭐⭐☆ / ⭐⭐⭐☆☆

---

### 候補2: ...

---

## 既存 Skill で対応可能なもの

- {トピック} → `.claude/skills/{skill-name}/SKILL.md` で対応済み

## Skill化不要と判断したもの

- {トピック}: {理由}
```

## 5. 確認

提案を私に提示し、Skill を作成すべきか確認を求めてください。

承認された場合は、以下の構成で Skill を作成してください：

```
.claude/skills/{skill-name}/
└── SKILL.md
```

**Skill ファイルに含める内容:**
- 概要
- プロジェクト構成（該当する場合）
- 実装パターン/コード例
- 設定方法
- チェックリスト

**注意:** 承認なしに Skill を作成しないでください。
