# 一般ルール

このプロジェクト全体に適用されるルール。

## 開発フロー

1. 作業開始前に `docs/PLAN.md` で対象タスクを確認する
2. 関連する要件定義（`docs/requirements/*.md`）を確認する
3. 1タスク1機能で進める（複数タスクを同時に進めない）
4. 作業完了後に `docs/PLAN.md` のステータスを更新する

## コミット規約

コミットメッセージは以下の形式で記述する：

```
<type>: <subject>

<body>
```

**type:**
- `feat`: 新機能
- `fix`: バグ修正
- `docs`: ドキュメントのみの変更
- `refactor`: リファクタリング
- `test`: テストの追加・修正
- `chore`: ビルド、設定等の変更

**例:**
```
feat: セッション作成APIを実装

- POST /api/kernels エンドポイントを追加
- カーネル起動処理を実装
```

## 禁止事項

- 要件定義を確認せずにコードを書き始めること
- 複数タスクを同時に進めること
- テストなしで「完了」とすること（テストが定義されているタスクの場合）
