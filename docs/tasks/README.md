# タスク詳細ファイル

このディレクトリには、各タスクの詳細な開発計画を格納する。

## ファイル構成

```
docs/tasks/
├── README.md           ← このファイル
├── _template.md        ← テンプレート
├── 1.1.1-dockerfile.md ← タスクごとの詳細
├── 1.1.2-docker-compose.md
└── ...
```

## ファイル命名規則

```
{タスク番号}-{タスク名（kebab-case）}.md
```

例：
- `1.1.1-dockerfile.md`
- `1.2.4-execute-code-tool.md`
- `2.1.3-catalog-loader.md`

## 運用フロー

```
1. /custom-plan-task でタスク詳細を作成
   ↓
2. 人間がレビュー・承認
   ↓
3. /custom-start-task で実装開始
   ↓
4. /custom-complete-task で完了
```

## タスク詳細に含める内容

- **概要**: 何を実現するか
- **関連ドキュメント**: 参照すべき要件・API仕様
- **実装計画**: 作成するファイル、実装手順
- **技術的な考慮事項**: 注意点、設計判断
- **完了条件**: 何をもって完了とするか
- **テスト計画**: どうテストするか
