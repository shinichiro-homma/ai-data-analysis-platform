# タスク詳細: 1.2.1 package.json, tsconfig.json 作成

## 概要

jupyter-mcp プロジェクトの初期化として、package.json と tsconfig.json を作成する。
これにより TypeScript による MCP サーバー開発の基盤が整い、後続タスク（1.2.2 jupyter-client 実装）に進むことができる。

## 関連ドキュメント

- 要件定義: `docs/requirements/jupyter-mcp.md` の「技術仕様」セクション
  - 技術スタック: TypeScript, MCP TypeScript SDK, axios
  - ディレクトリ構成
  - 起動コマンド（npm run dev, npm start 等）
- Skill: `.claude/skills/mcp-typescript-server/SKILL.md`
  - 依存関係、tsconfig.json 設定、package.json scripts

## 実装計画

### 作成するファイル

| ファイル | 内容 |
|----------|------|
| `jupyter-mcp/package.json` | プロジェクト設定、依存関係、スクリプト定義 |
| `jupyter-mcp/tsconfig.json` | TypeScript コンパイラ設定 |
| `jupyter-mcp/src/index.ts` | 最小限のエントリーポイント（動作確認用） |

### 実装手順

1. **package.json 作成**
   - プロジェクト名: `jupyter-mcp`
   - type: `module`（ESM）
   - 依存関係:
     - `@modelcontextprotocol/sdk`: ^0.5.0（MCP SDK）
     - `axios`: ^1.6.0（HTTP クライアント）
   - 開発依存関係:
     - `@types/node`: ^20.0.0
     - `typescript`: ^5.0.0
   - スクリプト:
     - `build`: tsc
     - `start`: node dist/index.js
     - `dev`: tsc --watch
     - `typecheck`: tsc --noEmit

2. **tsconfig.json 作成**
   - Skill に記載の設定を使用
   - target: ES2022
   - module: NodeNext
   - moduleResolution: NodeNext
   - strict: true
   - outDir: ./dist
   - rootDir: ./src

3. **src/index.ts 作成（最小限）**
   - shebang (`#!/usr/bin/env node`) を含む
   - 起動確認用の console.error 出力のみ
   - 本格実装は後続タスクで行う

4. **動作確認**
   - `npm install` で依存関係をインストール
   - `npm run build` でビルド成功を確認
   - `npm start` で実行確認

### 技術的な考慮事項

- **ESM (ECMAScript Modules)**: MCP SDK は ESM なので、package.json に `"type": "module"` を設定
- **moduleResolution: NodeNext**: `.js` 拡張子付きのインポートが必要（`import { foo } from "./bar.js"`）
- **shebang**: MCP サーバーは CLI として実行されるため、エントリーポイントに `#!/usr/bin/env node` を追加
- **HTTP クライアント**: 要件定義では「axios または fetch」とあるが、axios を使用（より豊富な機能）
